-- Atividade 01 Web --

Descreva as motivações para o desenvolvimento de aplicações baseadas em internet ?

As motivações para o desenvolvimento de aplicações baseadas em internet giram em torno da quebra de barreiras geográficas e técnicas que o software tradicional impunha. Antigamente, a necessidade de instalar e configurar programas individualmente em cada máquina gerava um custo de manutenção altíssimo e limitava o acesso do usuário a um dispositivo específico. Com a evolução da web, o foco mudou para a portabilidade e a centralização, permitindo que qualquer pessoa com um navegador e conexão acesse ferramentas complexas de qualquer lugar do mundo. Além disso, o modelo web facilitou drasticamente o ciclo de atualização de software, pois os desenvolvedores podem implementar melhorias e correções de segurança no servidor de forma instantânea para toda a base de usuários, eliminando a dependência de atualizações manuais por parte dos clientes.

Quais as funções de um servidor de aplicação? 

O servidor de aplicação desempenha um papel fundamental como o intermediário inteligente entre o banco de dados e o usuário final. Suas funções consistem primordialmente em processar a lógica de negócio, que são as regras específicas que regem o funcionamento do sistema, como cálculos complexos, validações de dados e fluxos de decisão. Ele também é responsável por garantir a segurança da aplicação, gerenciando o controle de acesso e a autenticação para assegurar que informações sensíveis não sejam expostas indevidamente. Além disso, o servidor de aplicação gerencia a persistência de dados, comunicando-se com os bancos de dados para salvar e recuperar informações de maneira eficiente e organizada, servindo como uma camada de abstração que permite ao sistema escalar conforme a demanda aumenta.

Quais as diferenças das camadas de frontend e backend ? 

Às diferenças entre as camadas de frontend e backend, podemos compreendê-las pela localização do processamento e pelo objetivo final de cada uma. O frontend é a camada de apresentação, que executa no navegador do usuário e tem como foco principal a experiência humana e a interface visual, utilizando tecnologias que interpretam cores, fontes e interatividade imediata. Em contrapartida, o backend opera no lado do servidor e lida com tudo o que é invisível aos olhos do público, como a integração com servidores, a proteção de dados e a arquitetura de sistemas. Enquanto o frontend se preocupa em como a informação é exibida e capturada de forma amigável, o backend garante que essa informação seja processada corretamente, armazenada com segurança e enviada para o destino certo com a máxima performance.
